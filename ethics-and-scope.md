# Ethics, Scope, and Responsible Use

## Purpose of This Repository

This repository exists to **document attacker tradecraft in Azure environments** for the purpose of:

- Education
- Authorised penetration testing
- Red team skill development
- Defensive understanding of attacker behavior

It is **not** intended to enable unauthorised access, abuse, or harm.

---

## Scope Boundaries

All content in this repository adheres to the following boundaries:

### Included

- High-level attack logic and decision-making
- Abuse scenarios based on **misconfigurations and trust relationships**
- Conceptual explanations of attacker behavior
- Diagrams and flowcharts
- Lab-safe assumptions

### Excluded

- Live exploit weaponisation
- Credential harvesting techniques
- Phishing kits or social engineering scripts
- Malware payloads or loaders
- Step-by-step instructions targeting real tenants

---

## Assumed Authorisation

All scenarios assume:

- Explicit authorisation
- Lab or test Azure tenants
- Written permission (where applicable)
- Compliance with local laws and organisational policy

If you do not have permission to test an environment, **do not attempt to replicate these scenarios**.

---

## Responsible Disclosure

If vulnerabilities or misconfigurations are identified during research:

- Follow responsible disclosure processes
- Notify affected organisations appropriately
- Avoid public disclosure of sensitive details

---

## Ethical Red Teaming Philosophy

This project follows a **professional red team mindset**:

> Understand how attackers think  
> Document how abuse occurs  
> Improve defensive posture through knowledge

Offensive knowledge without ethics is irresponsible.  
Ethical red teaming strengthens security for everyone.
